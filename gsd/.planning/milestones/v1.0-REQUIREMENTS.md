# Requirements: GymCoach

**Defined:** 2026-03-06
**Core Value:** Log workouts quickly at the gym and see what weight to lift next based on past performance

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Exercises

- [x] **EXER-01**: User can browse a library of exercises categorized by muscle group and equipment
- [x] **EXER-02**: User can create custom exercises with name, muscle group, and equipment

### Programs

- [x] **PROG-01**: User can create a workout program as an ordered list of exercises with target sets/reps/weight
- [x] **PROG-02**: User can edit and delete programs

### Logging

- [x] **LOG-01**: User can start a workout from a program template
- [x] **LOG-02**: User can log each set with actual weight and reps (pre-filled from last session)
- [x] **LOG-03**: User can edit or delete any logged set, exercise, or workout
- [x] **LOG-04**: User sees a rest timer that auto-starts after logging a set (configurable duration)

### History

- [x] **HIST-01**: User can view chronological list of completed workouts with exercises and sets
- [x] **HIST-02**: User can view per-exercise history showing recent sessions with weight/reps

### Progression

- [x] **PRGS-01**: User sees auto-suggested weight for next session based on past performance
- [x] **PRGS-02**: Progression algorithm uses simple linear model (hit all target reps → increase weight)

### Scripts

- [x] **SCRP-01**: DB schema migrations manage SQLite schema changes
- [x] **SCRP-02**: Seed script populates exercise library (research open-source exercise databases)
- [x] **SCRP-03**: CLI analytics show 1RM trends, volume per muscle group, workout frequency
- [x] **SCRP-04**: Export/import scripts for JSON/CSV data backup

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### UI Polish

- **UI-01**: Mobile-responsive design with large tap targets for gym use
- **UI-02**: Workout duration tracking (auto-start/stop timer)
- **UI-03**: Workout notes (per-workout and per-exercise free text)

### Advanced Progression

- **PRGS-03**: Deload suggestions when progression stalls
- **PRGS-04**: Per-exercise progression rule configuration (different increments per exercise)

### Social / Advanced

- **ADV-01**: PR tracking and celebration (auto-detect personal records)
- **ADV-02**: Volume tracking and trends (weekly/monthly charts per muscle group)
- **ADV-03**: Superset/circuit support (grouped exercises with shared rest timer)

## Out of Scope

| Feature | Reason |
|---------|--------|
| User authentication / multi-user | Single-user personal tool, no login needed |
| Mobile native app | Web-only, mobile-responsive is sufficient |
| Social features / sharing | Personal tool, not a community app |
| Video / form tracking | Different product entirely, ML complexity |
| Cloud sync | SQLite-local is a constraint, no server infrastructure |
| Nutrition / calorie tracking | Separate domain, existing apps do this well |
| AI-generated programs | User is the coach, app just tracks and suggests weight |
| Cardio tracking | Different data model (duration/distance vs sets/reps/weight) |
| Wearable integration | Web-only per constraints |
| Complex periodization engine | Simple linear progression is sufficient for v1 |
