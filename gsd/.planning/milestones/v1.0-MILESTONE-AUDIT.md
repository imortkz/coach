### Integration Issues

#### INT-01: Import program_id mapping gap (Medium)

**File:** `../scripts/gymcoach_scripts/export_import/importer.py:124`
**Affects:** SCRP-04

The importer correctly remaps exercise IDs via `exercise_id_map` but does not remap program IDs for workout rows. After deleting and re-inserting programs, autoincrement IDs may differ from exported values, causing dangling FK references on `workout.program_id`. This would trigger FK constraint violations during import.

**Fix:** Build `program_id_map` analogous to `exercise_id_map` and apply when setting `workout.program_id`.

#### INT-02: rest_timer_disabled not settable from UI (Low)

**Files:** `../frontend/src/stores/programs.ts`, `../frontend/src/views/ProgramEditView.vue`
**Affects:** LOG-04

The backend supports a `rest_timer_disabled` flag on programs, and the frontend reads it during workouts. However, the program builder UI never sends this field -- all programs default to `rest_timer_disabled=false`. The feature exists end-to-end but cannot be toggled by the user.

**Fix:** Add toggle to ProgramEditView and include in ProgramCreatePayload.
