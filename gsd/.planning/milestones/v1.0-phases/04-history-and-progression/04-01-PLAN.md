    **Repo: ../backend/**

    **1. Add schemas** to `app/workouts/schemas.py`:
    - `SuggestionInfo(BaseModel)`: type ("weight"|"reps"|"keep"), suggested_weight_kg (float|None), suggested_reps (int|None), increment (float|None), reason ("hit_target"|"missed_reps")
    - `WorkoutListResponse(BaseModel)`: items: list[WorkoutRead]
    - `ExerciseSessionSet(BaseModel)`: set_number, weight_kg, reps, is_warmup
    - `ExerciseSession(BaseModel)`: date (datetime), sets (list[ExerciseSessionSet]), best_weight (float|None), total_volume (float)
    - `ExerciseHistoryResponse(BaseModel)`: sessions (list[ExerciseSession]), suggestion (SuggestionInfo|None)
    - Update `WorkoutStartResponse` to add: `suggestions: dict[int, SuggestionInfo] = {}`

    **2. Add progression function** in `app/workouts/routes.py`:
    - `compute_progression(db, exercise_id, program_id) -> SuggestionInfo | None`
    - Equipment increments dict: Barbell +2.5, Dumbbell +2.0, Cable +2.5, Machine +2.5, Bodyweight None (+1 rep), Kettlebell +4.0, Smith Machine +2.5, EZ Bar +2.5, Resistance Band None (+1 rep), Trap Bar +2.5, Landmine +2.5
    - Check user-overridden increments in Settings table with key pattern `progression_increment_{equipment_lower_underscored}`
    - Find last completed workout with sets for exercise, get non-warmup sets, check if ALL have reps >= target_reps at same weight
    - If all hit target and weight increment: suggest current_weight + increment
    - If all hit target and bodyweight/band: suggest target_reps + 1
    - If missed: suggest same weight with reason "missed_reps"
    - If mixed weights or no history: return None

    **3. Add paginated list endpoint** in `app/workouts/routes.py`:
    - `GET /api/workouts` (note: place BEFORE the `/{workout_id}` route to avoid path conflicts)
    - Query params: limit (default 20, max 100), offset (default 0), program_id (optional filter)
    - Filter: `completed_at IS NOT NULL`, order by `completed_at DESC`
    - Use `selectinload(Workout.sets).selectinload(WorkoutSet.exercise)` for eager loading
    - Also `selectinload(Workout.program)` -- NOTE: check if Workout model has program relationship. If not, add it: `program = relationship("Program")` and import Program in models.py
    - Response model: `WorkoutListResponse`

    **4. Update `start_workout`** to also compute suggestions:
    - After computing pre_fill, iterate program exercises and call `compute_progression(db, exercise_id, program_id)` for each
    - Add non-None results to `suggestions` dict keyed by exercise_id
    - Include in WorkoutStartResponse

    **5. Add exercise history endpoint** in `app/exercises/routes.py`:
    - `GET /api/exercises/{exercise_id}/history`
    - Query params: limit (default 20), program_id (optional, for progression)
    - Find distinct completed workouts containing sets for this exercise, ordered by completed_at desc, limited
    - For each workout: extract sets for this exercise, compute best_weight (max weight_kg among non-warmup), total_volume (sum of weight_kg * reps for non-warmup)
    - If program_id provided, compute progression suggestion
    - Response model: `ExerciseHistoryResponse`

    - Add `TestListWorkouts` and `TestExerciseHistory` and `TestProgression` classes
    - Use existing conftest.py fixtures (client, db_session)
    - Create test data: program with exercises, completed workouts with sets
    - Test all behaviors listed above
    - Ensure warmup sets are created in test data to verify they're excluded from progression
