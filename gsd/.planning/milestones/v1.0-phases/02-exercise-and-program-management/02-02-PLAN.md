    - POST /api/programs with {name, exercises: [{exercise_id, order, sets: [{set_number, target_reps, target_weight_kg, is_warmup}]}]} creates program, returns 201 with full nested data
    - POST /api/programs with empty name returns 422
    - GET /api/programs returns list of all programs with nested exercises and sets (eager loaded, no N+1)
    - GET /api/programs/:id returns single program with full nested data
    - GET /api/programs/:id with non-existent id returns 404
    - PUT /api/programs/:id replaces entire program (delete all old exercises/sets, create new ones from request body). Returns 200 with updated data
    - PUT /api/programs/:id with non-existent id returns 404
    - DELETE /api/programs/:id removes program (cascade deletes exercises and sets). Returns 204
    - DELETE /api/programs/:id with non-existent id returns 404
    - Same exercise_id can appear multiple times in one program (different order values)
    - Program with 3 exercises, each with 3-4 sets including warmup sets, round-trips correctly (create then GET returns same structure)
  </behavior>
