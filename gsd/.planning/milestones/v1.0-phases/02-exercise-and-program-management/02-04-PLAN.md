  Build `../frontend/src/views/ProgramEditView.vue` -- a single component used for both create (/programs/new) and edit (/programs/:id/edit) modes.

  **Mode detection:**
  - If route has `:id` param, it's edit mode: fetch existing program via store.fetchProgram(id) and pre-populate form
  - If no `:id`, it's create mode: start with empty form

  **Program name:**
  - Text input at the top for program name. Required field.

  **Exercise list (the builder):**
  - Display added exercises as an ordered list. Each exercise row shows:
    - Exercise name + equipment tag
    - Up/down arrow buttons for reordering (disabled at top/bottom respectively)
    - Remove button (X or trash icon)
    - Expandable set configuration area (always visible by default)

  **Per-set targets (per exercise):**
  - Below each exercise, show a list of sets:
    - Each set row: "Set N" label, reps input (number), weight input (number, kg), warmup checkbox
    - "Add Set" button below the sets list
    - Remove set button (X) on each set row
  - Default when adding a new exercise: 3 sets with 10 reps each, no weight, not warmup
  - User example from CONTEXT: "dumbbell press, 12x20kg (warmup), 12x40kg, 12x45kg, 12x45kg"

  **Exercise picker:**
  - "Add Exercise" button at the bottom of the exercise list
  - Clicking it shows an inline search/autocomplete area (Claude's discretion on exact UX):
    - Text input to search exercises by name
    - Dropdown/list of matching exercises from the exercises store (fetch all exercises on mount, filter client-side)
    - Clicking an exercise adds it to the program's exercise list
    - Same exercise can be added multiple times
  - Keep the picker simple -- inline search with results list is sufficient

  **Reordering:**
  - Up arrow on an exercise: swap with the exercise above
  - Down arrow: swap with the exercise below
  - After any reorder, renumber all `order` values sequentially (1, 2, 3...)

  **Save:**
  - "Save Program" button at bottom
  - On save: construct the ProgramCreatePayload from form state. Set set_number values sequentially per exercise (1, 2, 3...). Set order values from position in list.
  - Create mode: call store.createProgram(), on success router.push('/programs')
  - Edit mode: call store.updateProgram(id, ...), on success router.push('/programs')
  - Show validation errors: program name required, at least 1 exercise required, each exercise needs at least 1 set

  **Cancel:**
  - "Cancel" button that navigates back to /programs

  **Style:** Same clean minimal aesthetic. Subtle borders between exercises. Compact set rows. Muted colors for labels. Blue primary button for save. Gray for cancel.
