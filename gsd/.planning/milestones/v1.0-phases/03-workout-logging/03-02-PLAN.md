    **Step 1: Expand TypeScript types**

    In ../frontend/src/types/index.ts, add:
    - `PreFillSet` interface: `{ set_number: number, weight_kg: number | null, reps: number | null, is_warmup: boolean }`
    - `WorkoutStartResponse` interface: extends Workout concept, adds `pre_fill: Record<number, PreFillSet[]>`

    - Add `exercise?: Exercise` to `WorkoutSet` if not already present (it is -- confirm)

    **Step 2: Create Pinia workout store**

    Create ../frontend/src/stores/workouts.ts following the composition API pattern from programs.ts:

    State refs:
    - `activeWorkout: ref<Workout | null>(null)` -- the current in-progress workout
    - `preFill: ref<Record<number, PreFillSet[]>>({})` -- pre-fill data from last session, keyed by exercise_id
    - `loading: ref(false)`
    - `error: ref<string | null>(null)`

    Methods:
    - `fetchActiveWorkout()`: GET /api/workouts/active. If 200, set activeWorkout + return true. If 404, set activeWorkout to null + return false. This is called on /workout page load.
    - `startWorkout(programId: number)`: POST /api/workouts with { program_id }. Set activeWorkout from response, set preFill from response.pre_fill. Return the workout.
    - `logSet(data: { exercise_id, set_number, weight_kg, reps, is_warmup })`: POST /api/workouts/{id}/sets. Append returned set to activeWorkout.sets (optimistic-style append, not full re-fetch per research pitfall 2). Return the created set.
    - `updateSet(setId: number, data: { weight_kg?, reps? })`: PUT /api/workouts/{id}/sets/{setId}. Update the set in activeWorkout.sets locally with the response.
    - `completeWorkout()`: PATCH /api/workouts/{id}/complete. Set activeWorkout.completed_at from response. Return workout.
    - Expose: `activeWorkout, preFill, loading, error, fetchActiveWorkout, startWorkout, logSet, updateSet, completeWorkout`

    Note: deleteSet, deleteExerciseSets, discardWorkout will be added in Plan 03 (they require undo toast).

    **Step 3: Build ProgramPicker component**

    Create ../frontend/src/components/workout/ProgramPicker.vue:
    - Uses `useProgramsStore` to fetch and display programs
    - Shows a list/grid of program cards with name and exercise count
    - Emits `select(programId: number)` when user taps a program
    - Shows loading state while fetching
    - Shows empty state if no programs exist ("Create a program first" with link to /programs)
    - Follows Linear/Notion minimal aesthetic from Phase 2

    **Step 4: Update WorkoutView.vue**

    Replace the stub in ../frontend/src/views/WorkoutView.vue:

    - `onStartWorkout(programId)`: calls `workoutsStore.startWorkout(programId)`, which transitions to the active workout view
