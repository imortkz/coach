    **Step 1: Create useRestTimer composable**

    Create ../frontend/src/composables/useRestTimer.ts:
    - State: `remaining` (ref number, seconds left), `isRunning` (ref boolean), `endTime` (number | null -- Date.now() target for tab-switch accuracy)
    - `start(seconds?: number)`: set endTime = Date.now() + (seconds ?? defaultSeconds) * 1000, start setInterval(1000) that computes remaining from endTime. On reaching 0: call stop() then notifyRestComplete(). Request notification permission on first call if not yet granted (`Notification.requestPermission()`).
    - `skip()`: call stop(), clear state.
    - `stop()`: clearInterval, isRunning = false, remaining = 0, endTime = null.
    - `notifyRestComplete()`: if Notification.permission === 'granted', fire `new Notification('Rest Complete', { body: 'Time for next set!' })`.
    - Tab-switch handling: listen to `document.addEventListener('visibilitychange', ...)`. On visible: if endTime exists and endTime <= Date.now(), trigger stop + notify immediately. Otherwise recompute remaining = Math.ceil((endTime - Date.now()) / 1000).
    - Clean up interval and event listener in the composable's onUnmounted/onScopeDispose.
    - Accept `defaultSeconds` parameter (number).
    - Return: `{ remaining, isRunning, start, skip }`

    **Step 2: Create RestTimer component**

    Create ../frontend/src/components/workout/RestTimer.vue:

    - Shows: remaining time as "M:SS" format, a progress bar (width shrinks as time passes), and a "Skip" button.
    - Emits: `skip` when skip button tapped.
    - Transition: slide up when isRunning becomes true, slide down when false (CSS transition or Tailwind transition classes).
    - Styling: semi-transparent dark background (bg-gray-900/90 text-white), rounded top corners, full width on mobile, max-w-2xl centered on desktop.
    - Only renders when isRunning is true (v-if or v-show with transition).

    - `discardWorkout()`: DELETE /api/workouts/{id}. Set activeWorkout to null.
    - `fetchRestTimerSetting()`: GET /api/settings/rest_timer_seconds. Return number (parse value string to int) or default 120 if 404.

    - Check if the program has `rest_timer_disabled` -- if so, do NOT start timer after set completion.
    - When a set is logged (after logSet completes successfully), call `restTimer.start()` unless timer disabled.
    - Render `<RestTimer>` component with timer state. Handle skip event.
    - Set up undo toast state: `undoToast: ref<{ message: string, undo: () => void } | null>(null)` and a `undoTimeoutId`. Render `<UndoToast>` bound to this state.
    - Provide undo toast functionality to child components via provide/inject or props/events. The undo pattern per research: for set delete, remove from local state immediately, show toast, setTimeout to call store.deleteSet(). If undo clicked, clearTimeout and restore local state.
