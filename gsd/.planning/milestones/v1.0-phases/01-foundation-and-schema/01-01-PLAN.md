  <action>
    **Repo: ../scripts/**

    1. Initialize the scripts repo with uv:
       ```bash
       cd ../scripts && uv init --app --bare
       uv add sqlalchemy alembic click rich python-dotenv
       uv add --dev ruff pytest
       ```

    2. Create `../data/` directory (shared DB location).

    3. Create `gymcoach_scripts/db/engine.py`:
       - Engine creation with `sqlite:///{DB_PATH}` where DB_PATH reads from `GYMCOACH_DB` env var, defaulting to `../../data/gymcoach.db` (relative to scripts repo root).
       - `connect_args={"check_same_thread": False}`.
       - Event listener on "connect" that runs `PRAGMA journal_mode=WAL` and `PRAGMA foreign_keys=ON`.
       - `SessionLocal = sessionmaker(bind=engine)`.

    4. Create `gymcoach_scripts/db/models.py` with ALL 6 models using SQLAlchemy 2.0 Mapped[] style:
       - **Exercise**: id (int PK), name (str unique), muscle_group (str), equipment (str), is_custom (bool default False)
       - **Program**: id (int PK), name (str), created_at (datetime)
       - **ProgramExercise**: id (int PK), program_id (FK programs.id), exercise_id (FK exercises.id), order (int), target_sets (int), target_reps (int), target_weight_kg (float nullable)
       - **Workout**: id (int PK), program_id (FK programs.id nullable), started_at (datetime), completed_at (datetime nullable)
       - **WorkoutSet**: id (int PK), workout_id (FK workouts.id), exercise_id (FK exercises.id), set_number (int), weight_kg (float nullable), reps (int nullable), is_warmup (bool default False)
       - **Setting**: key (str PK), value (str)
       - Include relationships: Program.exercises, ProgramExercise.program/exercise, Workout.sets, WorkoutSet.workout/exercise
       - Use cascade="all, delete-orphan" on Program.exercises and Workout.sets

    5. Create `gymcoach_scripts/cli.py` as Click group entry point.

    6. Create `gymcoach_scripts/migrate.py` as Click command that runs Alembic upgrade head programmatically.

    7. Initialize Alembic inside `gymcoach_scripts/db/alembic/`:
       - `alembic.ini` pointing to `gymcoach_scripts/db/alembic` for script_location
       - `env.py` that imports Base from models.py, sets target_metadata = Base.metadata, uses `render_as_batch=True`, and imports ALL model classes so autogenerate detects them
       - Configure sqlalchemy.url to use the same DB_PATH logic as engine.py

    8. Generate initial migration with `alembic revision --autogenerate -m "initial schema"` and run `alembic upgrade head`.

    9. Create `tests/conftest.py` with:
       - Fixture that creates an in-memory SQLite engine
       - Fixture that creates all tables via Base.metadata.create_all
       - Fixture that provides a Session for each test

    10. Create `tests/test_migrations.py`:
        - Test that running Alembic upgrade on a fresh DB creates all 6 tables (exercises, programs, program_exercises, workouts, workout_sets, settings)
        - Test that foreign keys are present (inspect table to verify FK columns exist)
        - Use a temporary SQLite file, not in-memory (Alembic needs file-based DB)

    **IMPORTANT per CLAUDE.md:** All file writes use absolute paths targeting /Users/vjkim/Code/TEST_GSD/scripts/. Git commit in scripts repo with tag [P1-01].
